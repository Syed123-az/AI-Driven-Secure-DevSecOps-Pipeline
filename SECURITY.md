# 🔐 Security Policy

## 📌 Supported Versions

This project is developed as part of an academic DevSecOps pipeline demonstration.
Currently, only the latest version available in this repository is actively maintained and supported.

| Version        | Supported |
| -------------- | --------- |
| Latest         | ✅ Yes     |
| Older Versions | ❌ No      |

---

## 🚨 Reporting a Vulnerability

If you discover a security issue in this project, please report it responsibly.

### 📩 How to Report:

* Create a **GitHub Issue** in this repository
* Clearly describe:

  * The vulnerability
  * Steps to reproduce
  * Expected vs actual behavior

---

## ⏱️ Response Time

* Initial response: **Within 48 hours**
* Resolution updates: Provided as needed

---

## 🔍 Scope of Security

This project focuses on:

* Detection of malicious payloads using Machine Learning
* Automated decision-making (ALLOW / BLOCK) in a DevSecOps pipeline

### ⚠️ Limitations:

* Does **not perform full static code analysis**
* Does **not replace enterprise-grade security tools**
* Designed for **educational and demonstration purposes only**

---

## 🛡️ Responsible Disclosure

Please do not publicly disclose vulnerabilities before they are reviewed and addressed.
This helps ensure responsible handling of potential security risks.

---

## 📢 Note

This project is intended for academic use and learning purposes.
Security mechanisms implemented are simplified and may not cover all real-world attack scenarios.
